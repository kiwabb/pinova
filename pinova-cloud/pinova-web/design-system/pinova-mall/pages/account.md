# Account page overrides

Read `../MASTER.md` first.

- Authentication is a quiet task surface, not a registration landing page.
- Keep one visual asset, one form, visible labels, inline errors and a single primary action.
- Login, registration and signed-in account states share the same route and visual shell.
- The signed-in state exposes only working destinations such as cart and shipping addresses.
- Destructive logout is visually separated from navigation actions.
