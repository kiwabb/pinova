# Pinova 教程路线

本目录存放面向开发者的 Pinova 教程。教程按业务理解顺序组织，不按前端、后端、数据库机械拆分。每篇教程都从用户场景出发，再沿调用链阅读各层代码。

## 推荐学习顺序

### 1. 项目地图与本地开发

介绍仓库结构、`pinova-cloud` 各模块职责、前后端启动方式和基础数据。它是后续教程的环境前置，但不与业务教程混写。

### 2. 商品、购物车与会员基础

介绍商品 SKU、会员身份、购物车和收货地址。订单创建依赖这些数据，先理解来源数据，再看订单如何消费它们。

### 3. 订单下单链路

当前首先编写的教材，完整追踪“购物车 → 确认订单 → 提交订单 → 订单落库”的一次请求：

- [订单下单链路](./orders/order-creation.md)

### 4. 支付与订单状态

在理解订单创建后，再介绍待支付、支付成功、零元订单、超时关闭和库存预占转换。

### 5. 会员订单查询

介绍“我的订单”如何按会员身份查询订单、订单明细和地址快照，以及前端状态筛选和分页。

### 6. 后台订单管理

最后介绍管理员身份、权限边界、订单列表和详情查询。后台教材建立在订单模型和会员订单查询已经清楚的基础上。

## 阅读方式

每篇教程优先回答三个问题：用户从哪里开始、请求经过哪些边界、数据最终如何形成业务事实。源码引用以当前仓库实现为准；领域约束和完整契约分别参考 `database/` 与 `.trellis/spec/` 下的文档。
