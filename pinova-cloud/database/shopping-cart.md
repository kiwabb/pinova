# 购物车表设计

购物车由 `pinova.shopping_cart` 和 `pinova.shopping_cart_item` 组成。主表保存购物车所有权和生命周期，购物车项保存用户对具体 SKU 的购买意图。

## 核心原则

- 一个会员同一时间只有一个活跃购物车。
- 游客通过高强度随机令牌识别，数据库只保存令牌的 SHA-256 十六进制哈希；原始令牌只进入安全 Cookie。
- 一个购物车内同一 SKU 只有一行，重复加入通过原子更新数量实现。
- 加入购物车不锁定价格、不预占库存。结算时必须重新校验 SKU 状态、实时价格、购买限制和可售库存。
- 购物车可以包含多个店铺的商品，结算阶段按 `shop_id` 拆分订单。
- 购物车属于交易边界，`shop_id`、`spu_id`、`sku_id` 是跨域逻辑引用，不建立到商品和商家表的数据库外键。

## 购物车主表

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `bigint` | 应用生成的购物车主键 |
| `member_id` | `bigint` | 登录会员主键，与游客令牌哈希二选一 |
| `guest_token_hash` | `varchar(64)` | 游客令牌 SHA-256 哈希，不保存原始令牌 |
| `status` | `smallint` | 购物车生命周期状态 |
| `merged_into_cart_id` | `bigint` | 游客购物车合并后的目标购物车 |
| `expires_at` | `timestamptz(3)` | 计划过期时间，游客购物车必须填写 |
| `last_activity_at` | `timestamptz(3)` | 最近有效操作时间，用于过期清理 |
| `closed_at` | `timestamptz(3)` | 合并、结算或过期关闭时间 |
| `version` | `integer` | 跨设备并发修改使用的乐观锁版本 |
| 审计字段 | - | 遵循数据库固定字段规范 |

购物车不使用软删除。它通过明确状态关闭，超过保留期后可以由清理任务物理删除；订单已经保存独立的成交快照，不能依赖购物车作为订单历史。

### 状态

| 值 | 名称 | 说明 |
| --- | --- | --- |
| `0` | 活跃 | 可以增删改商品并发起结算 |
| `1` | 已合并 | 游客登录后已合并进会员购物车 |
| `2` | 已结算 | 已成功创建订单，不再允许修改 |
| `3` | 已过期 | 游客或长期未活动购物车已关闭 |

部分唯一索引保证每个会员或游客令牌只有一个活跃购物车。跨行规则，例如合并目标必须是同一会员的活跃购物车，由 Service 在事务中校验。

## 购物车项表

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `bigint` | 应用生成的购物车项主键 |
| `cart_id` | `bigint` | 所属购物车，数据库外键 |
| `shop_id` | `bigint` | 商品所属店铺，用于结算拆单 |
| `spu_id` | `bigint` | 商品 SPU 主键，逻辑引用 |
| `sku_id` | `bigint` | 商品 SKU 主键，逻辑引用 |
| `quantity` | `bigint` | 期望购买数量，必须大于 0 |
| `selected` | `boolean` | 是否参与当前结算 |
| `version` | `integer` | 数量并发更新使用的乐观锁版本 |
| 审计字段 | - | 加入和最后修改时间 |

购物车项不保存商品名称、图片、规格、价格和库存快照。查询购物车时由应用层批量调用商品查询接口补齐展示数据，不能逐项发起远程调用。商品服务不可用时可以返回购物车基础状态和明确的暂不可校验状态，不能使用旧价格直接结算。

移除购物车项使用物理删除。购物车是临时购买意图，不需要为每次移除保留软删除记录；需要行为分析时由埋点或事件系统记录 `CartItemRemoved`，不能依赖在线表历史。

## 并发与结算

- 新增同一 SKU 使用 `INSERT ... ON CONFLICT (cart_id, sku_id) DO UPDATE`，并递增数量和版本。
- 修改数量必须携带购物车项旧 `version`，避免多个设备互相覆盖。
- 每次有效操作同步更新购物车 `last_activity_at` 和 `version`。
- 游客登录合并时，在同一事务内按 SKU 合并数量，再把游客购物车状态改为已合并。
- 发起结算时只处理 `selected = true` 的项，批量校验商品、价格、库存和限购规则。
- 库存预占成功并创建订单后，购物车才进入已结算状态；失败时购物车保持活跃。
- 订单中的商品名称、规格、单价和优惠必须保存订单快照，不能引用购物车项作为成交事实。

优惠券选择、促销选择、配送地址和结算试算结果不进入购物车主表。它们分别属于营销、履约和结算上下文，避免长期购物车保存已经过期的临时计算结果。

订单成交快照、拆单幂等和地址复制规则见 [`trade-order.md`](trade-order.md)。

## 初始化

建表脚本位于 [`database/init/020-create-shopping-cart.sql`](init/020-create-shopping-cart.sql)：

```bash
docker compose exec -T postgres \
  psql -v ON_ERROR_STOP=1 -U pinova -d pinova \
  < database/init/020-create-shopping-cart.sql
```
