# 星期八管理后台

## 本地运行

```bash
npm install
npm run dev
```

默认地址：`http://127.0.0.1:5173/`

## 管理员配置

1. 打开 CloudBase 环境 `cloud1-d6gtwjvgqe576cbf0`。
2. 在“身份认证 / 登录方式”中启用用户名密码登录并关闭公开注册。
3. 在控制台创建管理员用户名密码账号并复制 UID。
4. 在 `admins` 集合中新建同 UID 的文档：

```json
{
  "name": "店主",
  "role": "owner",
  "active": true
}
```

`owner` 可修改运营规则、会员资料和账号状态，也可永久删除用户作品。`editor` 可管理图集、教程和预约，但不能执行店主操作。

## 功能模块

- 运营概览：用户、会员、图集、作品、预约与近期到店安排
- 图集管理：封面、系列图纸、等级、排序、草稿、发布、下架和归档
- 预约管理：筛选、确认、取消和容量受控恢复
- 用户会员：会员结构、成长值、积分和账号状态
- 用户作品：预览、来源追踪和店主删除
- 教程内容：章节、图片、步骤、安全提醒、草稿和发布
- 运营配置：V1–V4 门槛与权益、积分奖励、门店实景图、门店信息和预约规则
- 操作日志：店主可追踪图集、预约、会员、作品、教程和运营配置的关键变更

所有敏感操作都由 `adminService` 再次校验管理员 UID；前端隐藏按钮不作为权限边界。

预约、用户和作品列表按 50 条分页加载。搜索和筛选作用于已加载记录；需要跨全部历史记录筛选时，继续点击“加载更多”直至全部载入。对应 CloudBase 复合索引见项目根目录 `README.md`。

## 检查与构建

```bash
npm run typecheck
npm run build
```

不要在前端代码中写入管理员密码、AppSecret 或支付密钥。
